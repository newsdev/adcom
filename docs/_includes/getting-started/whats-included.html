What's included

Adcom is downloadable in two forms, within which you'll find the following directories and files, logically grouping common resources and providing both compiled and minified variations.

jQuery required

Please note that all JavaScript plugins require jQuery to be included. Consult our bower.json to see which versions of jQuery are supported.

Bootstrap not required

While bootstrap enhances many of the features of Adcom, it is not required. The interactions defined by the JavaScript plugins were designed to work without any specific front end framework.

Precompiled Adcom

Once downloaded, unzip the compressed folder to see the structure of (the compiled) Adcom. You'll see something like this:

{% highlight bash %} adcom/ ├── css/ │ ├── adcom.css │ ├── adcom.min.css └── js/ ├── adcom.js └── adcom.min.js {% endhighlight %}

This is the most basic form of Adcom: precompiled files for quick drop-in usage in nearly any web project. We provide compiled CSS and JS (adcom.*), as well as compiled and minified CSS and JS (adcom.min.*). Fonts from Glyphicons are included, as is the optional Adcom theme.

Adcom source code

The Adcom source code download includes the precompiled CSS and JavaScriptassets, along with source Less, JavaScript, and documentation. More specifically, it includes the following:

{% highlight bash %} adcom/ ├── less/ ├── js/ ├── dist/ │ ├── css/ │ └── js/ └── docs/ └── examples/ {% endhighlight %}

The less/ and js/ are the source code for our CSS and JS (respectively). The dist/ folder includes everything listed in the precompiled download section above. The docs/ folder includes the source code for our documentation, and examples/ of Adcom usage. Beyond that, any other included file provides support for packages, license information, and development.
